## Краткое описание системы

**Фабрика гипотез** — интеллектуальный инструмент для генерации, обоснования и приоритизации научно-исследовательских гипотез в области материаловедения, химии и смежных дисциплин. Система предназначена для использования в НИИ, промышленных лабораториях и R&D-центрах для ускорения инновационного цикла и повышения качества принимаемых решений.

### Основные функции

- **Генерация гипотез** на основе целевого KPI, ограничений и контекстных данных (база знаний, интернет-источники).
- **Обоснование каждой гипотезы** цепочкой доказательств с привязкой к источникам (DOI, URL, патенты, внутренние отчеты).
- **Расчет бизнес-показателей**: ROI, окупаемость, стоимость НИОКР, риск-скорректированная NPV.
- **Визуализация знаний** в виде интерактивного графа связей между материалами, параметрами, KPI и механизмами.
- **Управление проектами** с автоматическим формированием чек-листов, сценариев, рисков и дорожных карт.
- **Интеграция с внешними источниками** (поиск в интернете, загрузка PDF/DOCX/TXT).
- **Прозрачность и интерпретируемость** — каждое решение сопровождается ссылками на источники и логикой вывода.

## Технический стек

| Компонент | Технологии |
|------------|------------|
| Бэкенд | Django 4.2, Django REST Framework |
| База данных | SQLite (по умолчанию), поддерживается PostgreSQL |
| Генеративные модели | Yandex AI Studio (YandexGPT) через SDK |
| Поиск в интернете | DuckDuckGo Search (ddgs) |
| Фронтенд | HTML5, Tailwind CSS, Vanilla JS (ES modules) |
| Граф знаний | Кастомная модель на Django (узлы, ребра) |
| Парсинг документов | PyPDF2, python-docx, встроенный текстовый парсер |
| Дополнительно | python-dotenv, requests, threading |

## Пошаговая инструкция по развертыванию

### 1. Клонирование репозитория

```bash
git clone https://github.com/your-username/POSVOLE_HYPOTHESIS_FACTORY)
cd POSVOLE_HYPOTHESIS_FACTORY
```

### 2. Выполнение скрипта запуска

Для ускорения запуска системы выполните одну команду (пропустив ручную установку зависимостей и миграций):

```bash
./run.bat
```

Для полного обнуления системы и запуска с «чистого листа» (удаление БД и всех миграций):

```bash
./run.bat reset
```

Если вы предпочитаете ручную настройку, следуйте шагам 3–9.

### 3. Создание и активация виртуального окружения

**Windows**

```bash
python -m venv venv
venv\Scripts\activate
```

**macOS/Linux**

```bash
python3 -m venv venv
source venv/bin/activate
```

### 4. Установка зависимостей

```bash
pip install -r requirements.txt
```

> Если вы планируете использовать GPU для работы с NLP-моделями (опционально), установите torch с поддержкой CUDA согласно инструкции.

### 5. Настройка переменных окружения

Создайте файл `.env` в корневой папке проекта и укажите следующие параметры:

```env
# Секретный ключ Django (сгенерируйте свой)
SECRET_KEY=django-insecure-your-secret-key-here

# Режим отладки (True для разработки, False для production)
DEBUG=True

# Ключи Yandex AI Studio (обязательны для работы генерации гипотез)
YANDEX_API_KEY=ваш_ключ_яндекс_ai
YANDEX_FOLDER_ID=ваш_folder_id
YANDEX_MODEL=yandexgpt  # или yandexgpt-lite

# Опционально: настройки базы данных (по умолчанию SQLite)
# DB_ENGINE=django.db.backends.postgresql
# DB_NAME=rd_engine
# DB_USER=postgres
# DB_PASSWORD=password
# DB_HOST=localhost
# DB_PORT=5432
```

### 6. Инициализация базы данных

```bash
python manage.py makemigrations core objects hypotheses knowledge graph
python manage.py migrate
```

### 7. Генерация API-токена для фронтенда

Скрипт создаст суперпользователя `admin` (пароль: `Turedy123`) и запишет токен в `static/js/token.js`:

```bash
python scripts/generate_token.py
```

### 8. Загрузка тестовых данных (опционально)

```bash
python scripts/seed_data.py
```

Это создаст:

- Объекты «Литий-ионный аккумулятор» и «Углеволоконный композит».
- Пример гипотезы с цепочкой доказательств.
- Тестовые узлы и связи графа.
- Документ в базе знаний.

### 9. Запуск сервера разработки

```bash
python manage.py runserver
```

После запуска откройте:

<http://127.0.0.1:8000>

**Учетные данные для входа:**

- Логин: `admin`
- Пароль: `Turedy123`

Админка:

<http://127.0.0.1:8000/admin>

### 10. Загрузка научных статей через скрипт (опционально)

```bash
python scripts/run_ingestion.py --doi 10.1016/j.matlet.2024.1352
```

или

```bash
python scripts/run_ingestion.py --all
```

## Пользовательская инструкция

После успешного запуска системы пользователь получает доступ к веб-интерфейсу с шестью основными разделами.

### 4.1. Панель управления

Отображает:

- Топ-1 стратегическую рекомендацию.
- Канбан-доску статусов гипотез.
- Объекты исследования.
- Статистику базы знаний.

### 4.2. Объекты и устройства

Позволяет:

- создавать исследуемые объекты;
- задавать KPI;
- моделировать сценарии What-If;
- получать рекомендации;
- создавать гипотезы.

### 4.3. Проекты

Автоматически формируются:

- чек-листы;
- дорожные карты;
- анализ рисков;
- бизнес-метрики;
- аналогичные проекты.

### 4.4. Реестр гипотез

Позволяет генерировать гипотезы по:

- объекту исследования;
- целевому KPI;
- ожидаемому улучшению;
- бюджету;
- срокам;
- параметрам команды.

После генерации система формирует:

- обоснование;
- цепочку доказательств;
- дорожную карту;
- ROI и другие показатели.

### 4.5. Граф связей

Поддерживает:

- масштабирование;
- поиск;
- перетаскивание узлов;
- обнаружение "белых пятен";
- просмотр связей.

### 4.6. База знаний

Позволяет:

- загружать PDF, DOCX, TXT и JSON;
- искать документы;
- выполнять интернет-поиск;
- автоматически извлекать граф знаний.

```text

      │
 REST API
      │
Backend (Django)
      │
YandexGPT

Knowledge Base + Internet Search
```

1. **core**
2. **objects**
3. **hypotheses**
4. **knowledge**
5. **graph**
6. **strategy**

- Retrieval-Augmented Generation (RAG)
- Граф знаний
